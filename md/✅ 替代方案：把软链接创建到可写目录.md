因为 `/sys/class/` 是内核动态生成的目录树，不允许用户直接在里面创建新的子目录（即使是 root 也不行）。这是内核文件系统的保护机制。

## ✅ 替代方案：把软链接创建到可写目录

你完全可以把链接放在 `/tmp`、`/var/run` 或 `/root` 下，这些目录可写，而且重启后可以通过启动脚本重新创建。

### 临时创建（立即生效）
```bash
ln -s /sys/bus/iio/devices/iio:device2/in_illuminance_raw /tmp/light
cat /tmp/light   # 测试
```

### 永久创建（开机自动重建）

**方法一：在 `/etc/rc.local` 中添加**（确保 `/etc/rc.local` 可执行）
```bash
#!/bin/sh
ln -sf /sys/bus/iio/devices/iio:device2/in_illuminance_raw /tmp/light
exit 0
```

**方法二：创建一个 systemd 服务**（更可靠）
创建 `/etc/systemd/system/light-sensor-link.service`：
```ini
[Unit]
Description=Create symlink for light sensor
After=local-fs.target

[Service]
Type=oneshot
ExecStart=/bin/ln -sf /sys/bus/iio/devices/iio:device2/in_illuminance_raw /tmp/light
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target
```

然后启用：
```bash
systemctl enable light-sensor-link.service
systemctl start light-sensor-link.service
```

这样每次开机都会在 `/tmp/light` 创建软链接，你也可以把路径改成 `/var/run/light`。

## 📌 如果不依赖链接，更简单的方法

1. **环境变量**：
   
   ```bash
   export LIGHT_SENSOR=/sys/bus/iio/devices/iio:device2/in_illuminance_raw
   ```
   之后 `cat $LIGHT_SENSOR` 即可。
   
2. **别名**（在 `~/.bashrc` 中加入）：
   ```bash
   alias light='cat /sys/bus/iio/devices/iio:device2/in_illuminance_raw'
   ```

3. **直接使用原路径**，写个短脚本包装一下。

推荐使用 `systemd` 服务或 `/etc/rc.local` 创建软链接，这样既干净又符合你的预期。
